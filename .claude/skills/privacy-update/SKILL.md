프로젝트의 최근 변경사항을 분석해서 개인정보처리방침과 이용약관을 오늘 날짜 기준으로 업데이트해줘.

## 작업 순서

### 1단계: 변경사항 파악

아래 명령으로 최근 커밋 내역과 변경된 파일을 파악한다:

```bash
git log --oneline -20
git diff HEAD~5..HEAD --name-only
```

변경사항을 분석해서 다음 항목에 해당하는지 판단한다:

**개인정보처리방침 업데이트가 필요한 변경:**
- 새로운 데이터 수집 항목 추가 (카메라, 위치, 알림 권한 등)
- 새로운 제3자 서비스 연동 (새 API, 외부 SDK 추가)
- 기존 데이터 수집 방식 변경
- 새로운 기능으로 인해 수집 목적이 달라지는 경우

**이용약관 업데이트가 필요한 변경:**
- 서비스 기능 추가 또는 제거 (제5조 서비스 내용)
- 계정 관련 정책 변경 (탈퇴, 복구 등)
- 이용자 의무나 금지 행위 관련 변경

변경사항이 둘 다, 혹은 하나만 해당할 수도 있다. 판단 후 어떤 문서를 업데이트할지 먼저 설명한다.

### 2단계: 현재 최신 버전 파악

두 파일을 읽는다:
- `components/legal/PrivacyContent.tsx`
- `components/legal/TermsContent.tsx`

각 파일에서:
- 현재 `VERSIONS` 배열의 첫 번째 항목(최신 버전) 날짜 확인
- 최신 버전의 컴포넌트 함수 내용 확인

오늘 날짜가 이미 최신 버전과 같으면, 오늘 날짜 버전의 내용만 수정한다 (새 버전 추가 없이).
오늘 날짜가 기존 최신 버전보다 새로우면, 새 버전을 추가한다.

### 3단계: 오늘 날짜 계산

오늘 날짜를 `YYYY-MM-DD` 형식으로 구한다:
```bash
date +%Y-%m-%d
```

날짜 형식 예시:
- value: `"2026-04-15"`
- label: `"2026년 4월 15일"`
- 컴포넌트명: `V20260415` (날짜의 `-` 제거)

### 4단계: 파일 수정

업데이트가 필요한 각 파일에 대해 다음을 수행한다:

**새 버전 추가의 경우:**

1. 최신 버전 컴포넌트 함수를 복사해 새 날짜 이름으로 함수를 추가한다
2. 변경사항에 맞게 내용을 수정한다 (변경된 것만 반영, 나머지는 그대로)
3. `VERSIONS` 배열 맨 앞에 새 항목을 추가한다
4. `useState` 기본값을 새 날짜로 업데이트한다
5. 렌더링 조건(`{version === "..." && <V... />}`)을 추가한다

**기존 버전 수정의 경우:**

1. 해당 날짜의 컴포넌트 함수 내용만 수정한다

**코드 구조 참고 (PrivacyContent.tsx 기준):**

```tsx
const VERSIONS = [
  { value: "2026-04-15", label: "2026년 4월 15일" },  // 새로 추가
  { value: "2026-04-03", label: "2026년 4월 3일" },
  // ... 기존 버전들
] as const;

function V20260415() {  // 새로 추가
  return (
    <>
      {/* 최신 버전 내용을 기반으로 변경사항 반영 */}
    </>
  );
}

export function PrivacyContent() {
  const [version, setVersion] = useState<Version>("2026-04-15");  // 업데이트
  // ...
  {version === "2026-04-15" && <V20260415 />}  // 추가
  // ... 기존 조건들
}
```

### 5단계: 변경 내용 요약

작업 완료 후 다음을 보고한다:
- 오늘 날짜로 어떤 버전이 추가/수정됐는지
- 개인정보처리방침에서 달라진 내용 (구체적으로)
- 이용약관에서 달라진 내용 (구체적으로)
- 변경하지 않은 이유가 있다면 그것도 설명

## 주의사항

- 기존 버전 컴포넌트는 절대 수정하지 않는다 (히스토리 보존)
- 변경사항이 법적 문서에 영향이 없다고 판단되면 업데이트하지 않고 이유를 설명한다
- 내용은 기존 문체와 동일하게 유지한다 (존댓말, 법적 표현 방식)
- `TermsContent.tsx`의 이용약관 제5조(서비스의 내용)는 실제 서비스 기능 목록과 항상 일치해야 한다
