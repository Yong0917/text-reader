README.md 파일을 전체 프로젝트 기준으로 전면 리팩토링 해줘.

다음 요구사항을 반드시 반영해줘:

1. 프로젝트 전체 구조를 분석해서 아래 항목을 포함해 체계적으로 정리
   - 프로젝트 개요 (기획 의도, 목적, 핵심 기능)
   - 서비스 흐름 설명
   - 폴더 구조 분석
   - 주요 비즈니스 로직 설명

2. 사용된 기술 스택을 카테고리별로 정리
   - Frontend
   - Backend
   - Database
   - Infra / DevOps
   - Deployment
   - 기타 라이브러리
   각 기술을 왜 사용했는지도 간단히 설명 포함

3. 주요 화면을 분석하여
   - 기능 설명
   - 사용자 시나리오
   - 핵심 UI/UX 포인트
   - 실제 실행 화면 캡처 이미지 포함
     (이미지는 assets 폴더에 저장하고 README에 markdown 이미지로 삽입)

4. 단순 나열이 아니라
   - 포트폴리오 제출용 문서 수준으로
   - 기술적 깊이가 드러나도록

5. 문서 구조는 가독성 좋게
   - 목차 포함
   - 이모지 최소 사용
   - 마크다운 문법 정리
   - 코드 블록은 실제 코드 예시 포함

결과물은 바로 GitHub에 올려도 되는 완성도 높은 README 형태로 작성해줘.
